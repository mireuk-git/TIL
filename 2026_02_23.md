# 데이터의 정의
- 수/영상/단어 등의 형태로 된 의미 단위
- 관찰/실험/조사로 얻은 사실이나 자료
## 데이터의 특성
- 존재적 특성: 객관적 사실
- 당위적 특성: 추론/예측/추정/전망을 위한 근거
## 데이터 유형

- 정량적 데이터: 정형 데이터, 비용소모 적음

# 데이터와 정보
## DIKW 피라미드 모형
- `Wisdom`: 근본 원리에 대한 깊은 이해를 바탕으로 도출되는 `창의적 아이디어`
- `Knowledge`: 상호 연결된 `정보 패턴을 이해`하고 이를 토대로 예측한 결과물
- `Information`: 데이터의 `가공`, `상관관계` 간 이해를 통해 `패턴 인식`, `의미 부여`한 데이터
- `Data`: 가공 전의 순수한 수치 및 기호

위쪽에 위치할 수록 가치가 높아짐

# 데이터 기반의 지식경영
지식의 공유를 통해 문제해결 능력을 향상시키려는 기업경영방식
#### 지식의 형태
- `암묵지`: `언어로 설명 불가`, 개인의 경험이나 잠재적인 능력에서 비롯되는 지식
  - `주관적` 지식, `경험에 기반`을 둔 지식, `정황에 따라 달라지는` 지식
- `형식지`: `언어로 명료화`(문서화)되어 전달될 수 있는 지식
  - `코드화`된 지식, `체계화된 언어로 상호전달 가능한` 지식

## 지식 창조 모델(SECI)
암묵지와 형식지의 상호작용을 통해 이뤄지는 지적 창조 활동

1. 공통화(`Socialization`): 암묵적 지식을 다른 사람에게 `전달`(암묵지->암묵지)
2. 표출화(`Externalization`): 암묵적 지식을 책이나 교본 등의 `형식지로 가공` (암묵지->형식지)
3. 연결화(`Combination`): `책/교본에` 자신이 알고 있는 새로운 지식 `추가` (형식지->형식지)
4. 내면화(`Internalization`): 형식지를 보고 다른 직원들이 노하우 습득 (형식지->암묵지)

# 데이터베이스
- 여러 응용시스템들이 공용할 수 있도록 통합저장된 운영데이터의 집합
- `정보를 일원화`, 처리를 효율적으로 수행하기 위해 서로 관련성을 가지며 `중복이 없는` 데이터의 집합 유지

## 데이터베이스의 특징
- 통합데이터(Integrated): 똑같은 데이터가 원칙적으로 중복되어 있지 않음(`최소의,통제된 중복`)
- 저장데이터(Stored): 컴퓨터가 접근 가능한 `저장매체에 저장`
- 운영데이터(Operational): 한 조직의 고유한 `기능을 수행하기 위해 필요`
- 공용데이터(Shared): 한 조직의 여러 응용 프로그램이 `공동`으로 소유/유지/이용
- 변화하는 데이터(Changed): 새로운 데이터의 추가, 삭제, 갱신으로 항상 변화하면서도 정확함 유지

## 데이터베이스 설계 절차
1. 요구사항 분석
  - 데이터베이스 `용도 파악`
  - `요구사항명세서`
2. 개념적 설계
  - `개념적 구조` 설계
  - `E-R 다이어그램`
3. 논리적 설계
  - `정규화`
  - `논리적 스키마`
4. 물리적 설계
  - `물리적 구조 설계`
  - `물리적 스키마`
5. 구현
  - SQL문 작성, DBMS에서 데이터베이스 생성

## 데이터베이스 시스템(DBS)
데이터베이스에 데이터 저장, 저장된 데이터를 관리해 조직에 필요한 정보를 생성해주는 시스템
#### 구성요소
- `데이터베이스`: 여러 사용자가 `공유`해서 사용할 수 있도록 `통합`해서 `저장`한 운영데이터의 집합
- `DB 관리 시스템`: 응용프로그램 대신 DB에 존재하는 데이터의 검색/삽입/삭제/수정 및 DB를 공유하게 하는 시스템
- `데이터 언어`:DB를 구축하고 활용 및 관리하는 언어(DDL,DML,DCL)
- `사용자`: DB를 사용하기 위해 접근하는 모든 사람
- `하드웨어`: DB의 성능과 직결되는 인프라 환경

## 데이터베이스 관리시스템(DBMS)
파일시스템의 같은 내용의 데이터가 여러 파일에 중복 저장되거나 응용 프로그램이 데이터 파일에 종속되는 문제 해결

#### 발전과정
- 1세대(1960): 네트워크형, 계층형
- 2세대(70s): 관계형
- 3세대(80s): 객체지향형
- 4세대(90s): 객체관계형
- 5세대(00s): NoSQL(NotonlySQL)

# 데이터 분석
## 전통적 데이터 분석
- 조직 `내부` 데이터를 분석, `과거` 현상 기반
- 단순하고 소형화, 정형 데이터
- 특정 시스템 중심 분석
- 데이터 웨어하우스(DW)

### Data Warehouse(DW)
사용자의 의사결정 지원을 위해 축적한 데이터를 사용자 관점에서 주제별로 통합해 운영시스템과 사용자 사이의 별도의 장소에 저장해놓은 데이터베이스
- DW 구현 전에 설계(Schema on Write)
- 배치보고, BI(Businesss Intelligence) 및 시각화

- Data Mart(DM): 작은 규모를 가진 DW
#### 특징
- 주제중심
- 통합구조
- 시계열성(데이터가 시간에 따라 쌓임)
- 바휘발성(영속성)

## 빅데이터 분석
- `내부+외부` 데이터 활용해 분석, `현재` 사실을 기반으로 `미래 예측`
- 다양, 대용량, 정형/비정형 데이터 분석
- 복합 시스템 중심 분석
- 데이터레이크(Data Lake)

### Data Lake
수집된 정형/반정형/비정형 데이터를 변형없이 원형 데이터 그대로 저장
- 비관계형 및 관계형 데이터, 정형/반정형/비정형 데이터
- 분석 시에 쓰임(Schema on read)
- 기계학습, 예측분석, 데이터 디스커버리 및 프로파일링
