>06\. 객체지향 설계 원칙 중, `서브타입`(상속받은 하위 클래스)은 어디에서나 자신의 `기반타입`(상위 클래스)으로 교체할 수 있어야 함을 의미하는 원칙은?
>1. ISP(Interface Segregation Principle) 
>2. DIP(Dependency Inversion Principle)
>3. LSP(Liskov Substitution Principle) V
>4. SRP(Single Responsibility Principle)

객체지향 설계 원칙(SOLID)
- 단일 책임 원칙(SRP, Single Responsibility Principle): 클래스당 1책임
- 개방 폐쇄 원칙(OCP, Open-Closed Principle): 기존 코드를 수정하지 않고도 기능 확장
- 리스코프 치환 법칙(LSP, Liskov Substitution Principle): 하위클래스를 상위클래스로 교체
- 인터페이스 분리의 원칙(Interface Segregation Principle): 클라이언트 기준으로 `인터페이스` 분리
- 의존성 역전의 원칙(Dependency Inversion Principle): 자주 바뀌는 클래스에 의존하지 않기

>18\. 미들웨어 솔루션의 유형에 포함되지 않는 것은?
>1. WAS
>2. Web Server V
>3. RPC
>4. ORB

미들웨어의 종류
- 원격 프로시저 호출(RPC, Remote Procedure Call): 클라이언트가 원격에서 동작하는 프로시저 호출 
- 메시지 지향 미들웨어(MOM, Message Oriented Middleware): 메시지를 저장하면서 다른 업무를 지속할 수 있게 함
- ORB(Object Request Broker): `객체지향`, 객체 및 서비스를 요청하고 전송
- DB 접속 미들웨어: 애플리케이션과 `DB` 연결
- TP 모니터(Transaction Processing monitor): `C/S 시스템`, 분산 시스템의 애플리케이션 지원
- 웹 애플리케이션 서버(WAS, Web Application Server): `웹 애플리케이션` 지원
- 엔터프라이즈 서비스 버스(ESB, Enterprise Service Bus): 기업 안팎의 모든 시스템 환경 연동
