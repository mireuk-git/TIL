# Codekata
### 13959
[link](https://www.acmicpc.net/problem/13959)
#### python
```python
d=int(input())
r=int(input())
t=int(input())

r_sum=0
r_curr=4
t_sum=0
t_curr=4-d

while r_sum<r:
    r_sum+=r_curr
    r_curr+=1
    if t_curr>=3: t_sum+=t_curr
    t_curr+=1
    if t_sum>t:
        if t_sum-t == r-r_sum:
            break
print(r-r_sum)
```
- 나이를 기준으로 브루트포스 진행
- `r_sum`,`t_sum`: 각 상자의 현재까지의 초 누적합
- `r_current`,`t_current`: 더할 초 개수
- `t_sum`이 `t`보다 커져야 할 것이기 때문에 정지 조건의 검토는 `t_sum > t`부터 진행, 그 이전엔 빠른 진행을 위해 스킵
- 정지조건은 각각 두 상자의 현재까지의 누적합과 오류값의 차이가 같을 것

### LOC 기법
개발에 소요되는 인원, 자원, 기간 등으로 소프트웨어의 규모를 확인해 개발 계획 수립에 필요한 비용을 산정

#### 소프트웨어 비용 결정 요소
- 프로젝트 요소
  - 제품 복잡도
  - 시스템 크기
  - 요구되는 신뢰도
- 자원 요소
  - 인적 자원, 하드웨어 자원, 소프트웨어 자원
- 생산성 요소
  - 개발자 능력, 개발 기간

#### 비용 산정 기법
하향식 비용 산정 기법 - 과거의 유사한 경험을 바탕으로 비용 산정
  - 전문가 감정 기법 - 경험 많은 전문가들에게 비용 산정 의뢰
  - 델파이 기법 - 주관적인 편견 보완, 많은 전문가의 의견 종합
상향식 비용 산정 기법 - 세부적인 작업 단위별로 비용을 산정한 후 집계해 전체 비용 산정
  - LOC (원시 코드 라인 수, source Line of Code)기법
  - 개발 단계별 인원수(Effort per Task) 기법 - 기능 구현에 필요한 노력을 생명주기의 각 단계별로 산정
  - 수학적 산정 기법


#### LOC(source Line Of Code) 기법
각 기능의 원시 코드 라인 수의 비관치, 낙관치, 기대치로 예측치를 구하고 이를 이용해 비용 산정
 - 예측치 = (낙관치+4*기대치+비관치)/6

> 01\. 상향식 비용 산정 기법 중 `LOC(원시 코드 라인 수) 기법`에서 `예측치`를 구하기 위해 사용하는 항목을 쓰시오. 

낙관치, 기대치, 비관치

낙관치 - 가장 적은 값
기대치 - 현실적으로 걸릴 값
비관치 - 가장 높은 값
예측치 = (낙관치 + 기대치*4 + 비관치) / 6

> 02\. LOC 기법에 의해 `예측된 총 라인수가 36000라인`, 개발에 참여할 `프로그래머가 6명`, 프로그래머들의 평균 생산성이 `월간 300라인`일 때 개발에 소요되는 기간은?

36000/(6*300)=20
20개월

> 03\. LOC(Lines Of Code) 기법에 의해 예측된 프로젝트의 `총 라인 수가 80000LOC`이고, `투입 개발자 수는 8명`, 개발자 `1인당 월평균 생성이 500LOC`일 경우, 개발에 소요되는 시간을 쓰시오.

80000/(8*500)=20

> 04\. LOC 기법에 의해 예측된 `총 라인수가 50000라인`, 프로그래머의 `월 평균 생산성이 200라인`, 개발에 참여할 `프로그래머가 10인` 일 때, 개발 소요 시간은?

50000/(200*10)=25개월
