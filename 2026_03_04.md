# Codekata
## 11109
[link](https://www.acmicpc.net/problem/11109)
### python
```python
t=int(input())
for test_case in range(t):
    d,n,s,p=map(int,input().split())
    t1 = d+p*n
    t2 = n*s
    if t1 < t2: print("parallelize")
    elif t1 > t2: print("do not parallelize")
    else: print("does not matter")
```

### java
```java
import java.io.BufferedReader;
import java.io.IOException;
import java.io.InputStreamReader;
import java.util.StringTokenizer;

public class p11109 {
    public static void main(String args[])throws IOException{
        BufferedReader br = new BufferedReader(new InputStreamReader((System.in)));
        StringTokenizer st = new StringTokenizer(br.readLine());
        int t = Integer.parseInt(st.nextToken());
        for (int i = 0; i < t; i++) {
            st = new StringTokenizer(br.readLine());
            int d = Integer.parseInt(st.nextToken());
            int n = Integer.parseInt(st.nextToken());
            int s = Integer.parseInt(st.nextToken());
            int p = Integer.parseInt(st.nextToken());
            
            int t1 = n*s;
            int t2 = d+p*n;
            if (t1 > t2) System.out.println("parallelize");
            else if (t1 < t2) System.out.println("do not parallelize");
            else System.out.println("does not matter");
        }
    }
}
```

## 빅데이터
### 회의론
|원인|설명|진단|
|---|---|---|
|목적에 맞지 않은 `과도한 투자`|`도입만 하면` 모든 문제를 한번에 해소할 것임을 강조(공포 마케팅)|무엇을 분석하고 `어떻게 활용할 지`에 대한 준비 부족|
|`성과 위주`의 분석 프로젝트 수행|`기존` 시스템 분석 결과를 `빅데이터 분석 성공사례로` 과대 포장|`빅데이터가 불필요`한 분석 환경|
|`개인 정보 보호`에 대한 문제|데이터 수집과 활용 시 `개인정보 유출 문제 발생`|개인정보의 인지 부족과 부적합한 보안 정책 적용|

### 분석전략 인사이트
- 대규모 데이터 보유 만으로 가치를 창출할 거라는 오류 (크기X, 데이터 내용에서 `비즈니스 가치` 창출)
- 단순히 많은 분석이 경쟁우위를 확보할 것이라는 오류 (전략적인 `통찰력` 확보해 분석 추진, 핵심 비즈니스 및 분석 문제에 집중한 데이터 분석)
- `일차원적 분석`을 통해서도 효과를 확인 가능하나 `변화에 대응 어려움` (비즈니스 `기회 발굴` 및 `경영진의 지원`을 이끌어내기, 분석 활용 볌위 확대 및 전략과의 연계)
- 일차원적 분석: 하나의 요인 집중 분석
