# 정보처리기사
## 임계 경로(Critical Path)
방향 그래프에서 어떤 정점까지 도달하는 데 가장 오래 걸리는 경로
즉 어떤 경로도 임계경로보단 빠름

여기선 일정을 짤 때 걸리게 될 최대 기간을 의미함
WBS -> P/C -> 임계 경로 -> 간트차트

WBS(Work Breakdown Structure) - 기능들 소단위로 나열
P/C
임계 경로 - 프로젝트를 완료하는데 필요한 최대 시간(내게 필요한 최소 시간이 됨)
간트 차트

> 01\. CPM 네트워크가 다음과 같을 때 `임계경로의 소요 기일`을 쓰시오.
[사진 대체 링크](https://www.youtube.com/watch?v=_skT2N8cIcQ&list=PLniy99c_7ZfpURYCmquts3lQYWzFAIxod&index=32) 1:39

2일+3일+5일+4일=14일

모든 경로의 경우의 수를 다 합산하면서 나가는 게 좋다
단계마다 합산하며 진행하는게 좋을듯

2일+2일+3일+3일 = 10일
2일+3일+5일+4일 = 14일
3일+5일+4일 = 12일
10일 < 12일 < 14일

> 02\. 다음은 소작업 리스트에서 작업 `C`의 `가장 빠른 착수일`, `가장 늦은 착수일`, `여유기간(slack time)`을 순서대로 쓰시오.
|작업|선행작업|소요기간|
|A|-|15|
|B|-|10|
|C|A,B|10|
|D|B|25|
|E|C|15|

P/C 그리기

가장 빠른 착수일: 15(A)
가장 늦은 착수일: 15(A) (임계경로상에 포함되어 여유기간이 없음)
여유기간: 0 (임계경로상에 포함되어 여유기간이 없음)

가장 빠른 착수일: 모든 선행작업이 완료되고 바로 착수할 때
가장 늦은 착수일: 전체 임계경로 - 현재 트리의 임계경로
여유기간: 가장 늦은 착수일 - 가장 빠른 착수일
