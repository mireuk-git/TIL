```python
def solution(bandage, health, attacks):
    hp=health
    t=0
    for a in attacks:
        if hp<health:
            if a[0]-t>bandage[0]: hp+=min((a[0]-t-1)*bandage[1]+(a[0]-t-1)//bandage[0]*bandage[2],health-hp)
            else: hp+=min((a[0]-t-1)*bandage[1],health-hp)
            print(hp)
        hp-=a[1]
        if hp<=0: return -1
        t=a[0]
        print(hp)
    return hp
```
- 체력을 회복한 후 `hp`가 `health`보다 크다면 다시 `hp`를 `health`로 맞추는 방법으로 오버힐을 막을 수도 있겠다. 다만 그럴경우 중간에 커맨드를 악의적으로 우겨넣어 `hp`를 재세팅하는 걸 미뤄 오버힐되는 버그가 생길수도 있겠다.

### java
```java
class Solution{
    public int solution(int[] bandage, int health, int[][]attacks){
        int hp=health;
        int t=0;
        for (int[] a: attacks){
            if (hp<health) hp+=Math.min(health-hp, bandage[1]*(a[0]-t-1)+(a[0]-t-1)/(bandage[0])*bandage[2]);
            hp-=a[1];
            if (hp<=0) return -1;
            t=a[0];
        }
        return hp;
    }
}
```
- 연산순서는 계산결과에 영향을 미친다
