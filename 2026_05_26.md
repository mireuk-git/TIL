## 서버 증설 횟수
[link](https://school.programmers.co.kr/learn/courses/30/lessons/389479)
### python
```python
from collections import deque

def solution(players, m, k):
    server=deque()
    count=0
    for t in range(24):
        while len(server)>0 and server[0]<=t: server.popleft()
        while players[t]>=(len(server)+1)*m:
            count+=1
            server.append(t+k)
    return count
```
- `popleft()`가 있는 `deque`로 선입선출의 queue를 구현해 서버의 증설 및 반납 처리

### java
```java
import java.util.ArrayDeque;
import java.util.Queue;

class Solution{
    public int solution(int[] players, int m, int k){
        int count=0;
        Queue<Integer> servers = new ArrayDeque<>();
        for (int t=0;t<24;t++){
            while (!servers.isEmpty() && servers.peek()<=t) servers.poll();
            while (players[t]>=(servers.size()+1)*m) {
                count++;
                servers.offer(t+k);
            }
        }
        return count;
    }
}
```
- java의 `deque`는 맨 앞의 원소만 접근이 가능하도록 만들어진 자료구조
- 따라서 값 조회 메서드인 `peek()`로는 0번째 원소만 조회가 가능하다. 다른 인덱스의 값을 조회해야 한다면 `deque`말고 List를 고려해야 한다.
- `peek()`: queue의 가장 앞 원소 조회, queue의 인터페이스 특성상 다른 원소를 조회할 수 있는 기능은 없다
- `offer()`: queue의 끝에 값 추가
- `poll()`: queue의 앞에서 값 꺼내오기, return값은 삭제된 원소, 선입선출의 queue 특성상 오직 앞에서만 삭제
