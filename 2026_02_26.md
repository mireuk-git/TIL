# ADsP
## 빅데이터
- 데이터 양이 큰 데이터(수십 TB 이상)
- 대량의 정형/비정형 데이터 집합에서 가치를 추출하고 결과를 분석하는 기술

### 특징
3v
- Volume: 물리적 데이터가 겁나 큼(페타/엑사/제타)
- variety: 데이터의 다양성(정형/반정형/비정형)
- velocity: 실시간에 가까운 빠른 수집 및 처리

4v
- value: 비즈니스에 실현될 궁극적 가치

기타
- veracity
- visualization
- validity
- volatility

### 빅데이터가 만들어내는 변화
- 사전처리->사후처리: 정해진 특정 정보만 처리 -> 가능한 많은 데이터 모으고 다양한 방식으로 조합해 정보 찾아내기
- 표본조사->전수조사: 일부 샘플만 이용 -> 전체 데이터 이용해 전수조사
- 질->양: 당장 영향을 못줘도 다른 변수에 대해 풍부한 정보, 모든 데이터를 활용할 때 더 많은 가치 추출
- 인과관계->상관관계: 인과관계에 의한 미래예측 -> 데이터 기반의 상관관계분석

### 빅데이터 라이프사이클
수집 -> 저장 -> 처리 -> 분석
#### 수집
- ETL(Extract Transform Load) - 데이터를 추출해 비즈니스 데이터로 변환하는 방법(DW, DL)
- Crawling - 웹 로봇을 이용해 SNS, 뉴스, 웹 정보 등 인터넷 상의 웝 문서의 정보 수집
- Open API - API로 데이터를 수집, 소유 주체가 사용자를 위해 정보/데이터를 정해진 방식으로 공개
- Sqoop - 구조화된 데이터저장소(RDBMS)에서 데이터를 추출해 하둡의 HDFS에 저장 후 처리할 수 있도록 해주는 오픈소스 수집도구(정형)
- Kafka - 오픈소스 분산 발행/구독 메시징 시스템, 데이터 스트림을 실시간으로 관리, 대규모 트랜잭션 데이터가 발생했을 때 데이터를 버퍼링하면서 타겟 시스템에 안정적으로 전송

#### 저장
- RDBMS(Relational DataBase Managing System) - `관계형` 데이터 저장/수정/관리(Oracle,MS-SQL,MySQL,Sybase,MPP DB, Maria DB, PostgreSQL)
- NoSQL(Notonly SQL) - `비관계형` 데이터 저장소, `테이블 스키마`가 고정되지 않고 `수평적 확장` 용이
- 분산파일시스템(Distributed File System) - 여러 서버에 분산해 파일 저장,처리(GFS,HDFS)

#### 처리/관리
- Hadoop - 대용량 데이터 처리 분석, HDFS과 분산처리를 수행하는 MapReduce로 구현
  - MapReduce - 저장된 파일을 분산된 서버의 CPU와 메모리 자원 이용해 빠르게 분석 및 병렬 처리
  - HDFS(Hadoop Distributed File System) - 하둡 프로젝트의 분산파일시스템, 특정 크기로 파일 분산, 하나 이상의 보조 네임 노드, 다수의 데이터 노드로 구성

#### 분석/시각화
- 텍스트 마이닝 - `대규모 텍스트`로부터 의미있는 정보 추출, 분석(자연어처리)
- Machine Learning - 학습 데이터로부터 모델 생성, 대용량 데이터를 자동으로 분석, 귀납 추론
- Python - 객체지향, 명령형, 함수형 프로그래밍 스타일을 모두 지원하는 범용 프로그래밍 언어, 다양한 플랫폼에서 사용 가능(Glue Language), 풍부한 라이브러리(모듈) 제공
- SPSS(Statistical Package for the Social Sciences) - 사회과학 자료분석을 위해 고안된 통계분석 전용 프로그램
- R - 통계분석 기법, 알고리즘, 데이터 시각화 기능 지원 툴
- SAS(Statistical Analysis System) - 공인된 대부분의 통계분석, 보고서 작성, 그래픽 작업이 가능한 통계분석 프로그램
- Tableau - 시각적 분석 플랫폼(BI)
