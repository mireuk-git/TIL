>06\. `UML 확장 모델`에서 `스테레오 타입` 객체를 표현할 때 사용하는 기호로 맞는 것은?
>1. <<>> V
>2. (())
>3. {{}}
>4. [[]]

UML 확장 모델
구성요소 
- 사물(things): 관계가 형성될 수 있는 대상들
  - 구조(structual) 사물: 개념적, 물리적 요소, class, use case, component, node
  - 행동(behavioral) 사물: 요소들의 행위, 상호작용, 상태 머신 등
  - 그룹(grouping) 사물: 요소들을 그룹으로 묶어 표현(package)
  - 주해(annotion) 사물: 부가적인 설명/제약조건
- 관계(relationship)
  - 연관 관계 (->, --)
  - 집합 관계(<>-, 색칠안된마름모) 포함하는 쪽이 마름모
  - 포함 관계(<//>-, 색칠된마름모)
  - 일반화 관계(-|>, 색칠안된삼각형)
  - 의존 관계(점선)
  - 실체화 관계
- diagram
  - structural diagram
    - class diagram: 클래스의 속성, 관계
    - Object diagram: 객체와 객체의 관계
    - component diagram: 컴포넌트 간의 관계, 인터페이스, 구현단계에서 사용
    - deployment diagram: 결과물, 프로세스, 컴포넌트 등의 물리적 요소들의 위치 표현, 구현단계에서 사용
    - composite structure diagram: 복합 구조를 가지는 클래스 혹은 컴포넌트
    - package diagram: 모델 요소들을 그룹화한 패키지
  -Behavioral diagram
    - Usecase Diagram
    - Sequence Diagram
    - Communication Diagram
    - State Diagram
    - Activity Diagram
    - Interaction Overview Diagram
    - Timing Diagram

스테레오 타입: UML 기본에서 더 확장, <<>>으로 표현
